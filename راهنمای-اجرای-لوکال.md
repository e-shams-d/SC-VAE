# راهنمای اجرای لوکال پروژه‌ی SC-VAE (روی سیستم دارای کارت گرافیک)

این سند مرحله‌به‌مرحله توضیح می‌دهد که برای اجرای آموزش (Stage 1) مدل **SC-VAE** روی یک سیستم با کارت گرافیک NVIDIA چه پیش‌نیازهایی لازم است و چه کارهایی باید انجام شود. مطالب بر اساس بررسی مستقیم کدِ همین مخزن نوشته شده است.

---

## ⚠️ نکات مهمی که قبل از شروع باید بدانید

نتیجه‌ی بررسی سیستم فعلی و کد پروژه:

| موضوع | وضعیت | توضیح |
|------|--------|-------|
| کارت گرافیک این سیستم | ❌ ندارد | `nvidia-smi` پیدا نشد. این دستگاه فعلی برای اجرای GPU مناسب نیست؛ باید روی یک سیستم دارای کارت **NVIDIA** اجرا کنید. |
| conda روی این سیستم | ❌ نصب نیست | برای روش پیشنهادی باید نصب شود. |
| Python روی این سیستم | ✅ 3.10.11 | نسخه‌ی موردنیاز پروژه دقیقاً 3.10 است. |
| فایل `requirements.txt` | ❌ وجود ندارد | برخلاف چیزی که در `README.md` نوشته شده، چنین فایلی در مخزن نیست. به‌جای آن از `environment.yml` استفاده کنید. |
| نسخه‌ی PyTorch | `1.13.0 + CUDA 11.6` | نسخه‌ها پین‌شده‌اند و نباید عوض شوند (به‌خصوص `torchvision==0.14.0`). |
| دیتاست پیش‌فرض | **FFHQ** | باید دستی دانلود و در مسیر مشخص قرار گیرد؛ دانلود خودکار ندارد. |

## ۱) پیش‌نیازهای سخت‌افزاری

- **کارت گرافیک NVIDIA** با پشتیبانی CUDA (عملاً هر کارت نسبتاً جدید).
- **حافظه‌ی گرافیکی (VRAM):**
  - تصاویر ۲۵۶×۲۵۶ با `batch_size=8` (مقدار پیش‌فرض کانفیگ) سنگین است.
  - توصیه: **۱۲ گیگابایت یا بیشتر**. روی کارت‌های ۶–۸ گیگابایتی باید `batch_size` را به ۲ تا ۴ کاهش دهید.
- **رم سیستم:** حداقل ۱۶ گیگابایت.
- **فضای دیسک:** برای محیط conda چند گیگابایت + برای دیتاست FFHQ چندین گیگابایت (بسته به نسخه).

---

## ۲) پیش‌نیازهای نرم‌افزاری

- **سیستم‌عامل:** ویندوز (فایل `environment.yml` یک export ویندوزی است و بهترین سازگاری را روی ویندوز دارد) یا لینوکس (با کمی تطبیق دستی بسته‌ها).
- **درایور NVIDIA به‌روز:** سازگار با CUDA 11.6 (روی ویندوز نسخه‌ی درایور ≥ 452.39؛ عملاً هر درایور جدید کافی است).
  - 💡 **نیازی به نصب جداگانه‌ی CUDA Toolkit نیست.** بسته‌های pip مربوط به PyTorch، رانتایم CUDA را خودشان به‌همراه دارند؛ فقط **درایور** باید نصب باشد.
- **Python 3.10**
- **conda / Miniconda** (برای روش پیشنهادی نصب) یا `venv` (برای روش سبک).
- **git** (برای دریافت کد، در صورت نیاز).

برای بررسی سلامت درایور و کارت روی سیستم مقصد:

```powershell
nvidia-smi
```

اگر این دستور جدول کارت گرافیک را نشان داد، یعنی درایور درست نصب است.

## ۴) ساخت محیط با conda و نصب وابستگی‌ها (روش اصلی)

چون می‌خواهید با **conda** کار کنید، این بخش کامل و مرحله‌به‌مرحله توضیح داده شده است.

### ۴.۱) نصب Miniconda (روی این سیستم conda نصب نیست)

یکی از دو راه زیر را انتخاب کنید:

**الف) با winget (ساده‌ترین راه روی ویندوز):**
```powershell
winget install -e --id Anaconda.Miniconda3
```

**ب) نصب دستی:** نصب‌کننده را از این آدرس دانلود و اجرا کنید:
```
https://repo.anaconda.com/miniconda/Miniconda3-latest-Windows-x86_64.exe
```

پس از نصب، **«Anaconda Prompt»** را از منوی استارت باز کنید. اگر می‌خواهید conda در خودِ PowerShell هم کار کند، یک‌بار دستور زیر را اجرا کنید و سپس ترمینال را ببندید و دوباره باز کنید:
```powershell
conda init powershell
```

بررسی نصب:
```powershell
conda --version
```

### ۴.۲) ساخت محیط conda

> ✅ **اصلاح اعمال شد:** پیش‌تر فایل `environment.yml` آدرس مخزن مخصوص PyTorch (`--extra-index-url`) را نداشت و به همین دلیل اجرای `conda env create -f environment.yml` سرِ نصب `torch==1.13.0+cu116` با خطای **«No matching distribution found»** شکست می‌خورد. این خط اکنون به فایل اضافه شده، پس **راه ۲ مستقیماً کار می‌کند**. هر دو راه زیر معتبرند؛ راه ۱ سبک‌تر است.

#### راه ۱ — ساخت محیط تمیز + نصب با pip (پیشنهادی و مطمئن‌تر)

```powershell
# ۱) ساخت محیط با پایتون ۳.۱۰
conda create -n scvae python=3.10 -y

# ۲) فعال‌سازی محیط
conda activate scvae

# ۳) نصب PyTorch با CUDA 11.6 (نسخه‌ها را تغییر ندهید)
pip install torch==1.13.0+cu116 torchvision==0.14.0+cu116 torchaudio==0.13.0+cu116 --extra-index-url https://download.pytorch.org/whl/cu116

# ۴) نصب سایر وابستگی‌های لازم برای اجرای آموزش
pip install numpy==1.24.1 scipy scikit-image scikit-learn matplotlib opencv-python einops omegaconf easydict pyyaml tensorboardX tensorboard torchmetrics piq "pillow==9.2.0" tqdm six "protobuf==3.20.1"
```

> 💡 نیازی به نصب `cudatoolkit` از conda نیست؛ wheelهای pip مربوط به torch، رانتایم CUDA 11.6 را خودشان همراه دارند. فقط **درایور NVIDIA** باید نصب باشد.
>
> ⚠️ نسخه‌ی `torchvision` حتماً باید **0.14.0** بماند، چون کد از پارامتر قدیمی `range` در تابع `make_grid` استفاده می‌کند که در نسخه‌های جدیدتر حذف شده است.

#### راه ۲ — استفاده از خودِ `environment.yml` (محیط نویسندگان)

اگر می‌خواهید دقیقاً همان محیط نویسندگان (با نام `torch`) را بسازید. خط `--extra-index-url` لازم **از قبل به فایل اضافه شده**، پس کافیست مستقیم بسازید:

```powershell
conda env create -f environment.yml
conda activate torch
```

> توجه: این فایل buildهای پین‌شده‌ی ویندوزی دارد و سنگین‌تر است (شامل `tensorflow`، `clip` و `torch_geometric` که در کدِ آموزش **استفاده نمی‌شوند**). اگر فقط هدفتان اجرای آموزش است، **راه ۱ سریع‌تر و کم‌دردسرتر** است. همچنین این فایل مخصوص ویندوز است و روی لینوکس ممکن است بعضی نسخه‌های پین‌شده پیدا نشوند؛ در آن صورت از راه ۱ استفاده کنید.

### ۴.۳) بررسی صحت نصب GPU

پس از فعال‌سازی محیط:
```powershell
python -c "import torch; print('CUDA available:', torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'no gpu')"
```
خروجی باید `CUDA available: True` و نام کارت گرافیک باشد.

### (اختیاری) روش جایگزین بدون conda — venv + pip

اگر نخواهید از conda استفاده کنید، می‌توانید همان بسته‌های «راه ۱» را داخل یک محیط `venv` نصب کنید:


## ۵) آماده‌سازی دیتاست (مهم‌ترین مرحله)

دیتاست پیش‌فرض **FFHQ** است و کد آن را **به‌صورت خودکار دانلود نمی‌کند**.

### ساختار موردنیاز

- کد به‌طور پیش‌فرض دنبال تصاویر در این مسیر می‌گردد (نسبت به جایی که برنامه را اجرا می‌کنید):

  ```
  SC-VAE/dataset/FFHQ/resized/
  ```

- تصاویر باید **مستقیماً و به‌صورت تخت** داخل همین پوشه باشند، با نام‌هایی که در فایل‌های لیست آمده‌اند، مثلاً:

  ```
  dataset/FFHQ/resized/52535.jpg
  dataset/FFHQ/resized/59777.jpg
  ...
  ```

- فهرست نام‌ها از این فایل‌ها خوانده می‌شود:
  - آموزش: [img_datasets/assets/ffhqtrain.txt](img_datasets/assets/ffhqtrain.txt)
  - اعتبارسنجی: [img_datasets/assets/ffhqvalidation.txt](img_datasets/assets/ffhqvalidation.txt)

### گزینه‌ها برای تهیه‌ی داده

**گزینه ۱ — دیتاست کامل FFHQ:**
نسخه‌ی ۲۵۶×۲۵۶ تصاویر FFHQ را از مخزن رسمی [NVlabs/ffhq-dataset](https://github.com/NVlabs/ffhq-dataset) تهیه کنید و داخل `dataset/FFHQ/resized/` قرار دهید.

**گزینه ۲ — تعیین مسیر دلخواه:**
اگر تصاویر را جای دیگری دارید، در فایل کانفیگ زیرِ بخش `dataset` یک کلید `root` اضافه کنید (مرحله‌ی بعد توضیح داده شده).

**گزینه ۳ — آزمایش سریع با چند تصویر (بدون دانلود کل FFHQ):**
1. چند تصویر ۲۵۶×۲۵۶ دلخواه را در یک پوشه بگذارید.
2. نام همان فایل‌ها را در `ffhqtrain.txt` و `ffhqvalidation.txt` بنویسید (به‌جای محتوای فعلی).
3. مسیر `root` را به آن پوشه تنظیم کنید.

این راه برای «تستِ راه‌افتادن کل خط لوله» کافی است، هرچند برای آموزش واقعی مدل مناسب نیست.

> ℹ️ مسیر MNIST که در کد دیده می‌شود برای این اسکریپت ناقص است (متغیر `valid_dataset` تعریف نشده و تعداد کانال‌ها هم با کانفیگ FFHQ نمی‌خواند)؛ بنابراین برای اجرای واقعی روی **FFHQ** (یا ImageNet) تمرکز کنید.

## ۶) تنظیم فایل کانفیگ

کانفیگ پیش‌فرض: [configs/ffhq/stage1/ffhq256-scvae16x16.yaml](configs/ffhq/stage1/ffhq256-scvae16x16.yaml)

تنظیمات مهمی که ممکن است بخواهید تغییر دهید:

```yaml
dataset:
  type: ffhq
  root: ./dataset/FFHQ/resized   # ← در صورت نیاز این خط را اضافه/اصلاح کنید
  transforms:
    type: ffhq256x256

experiment:
  batch_size: 8     # ← اگر VRAM کم است به 2 یا 4 کاهش دهید
  epochs: 10        # ← برای تست سریع، 1 بگذارید
  smoke_test: False # ← True یعنی فقط ۱/۱۰ داده‌ها استفاده شود (تست سریع‌تر)
```

کانفیگ‌های دیگری هم برای اندازه‌های مختلف فضای latent موجودند (`scvae1x1`, `scvae4x4`, `scvae16x16`, `scvae32x32`) که با سوییچ `--model-config` انتخاب می‌شوند.

## ۷) اجرای آموزش با conda

**هر بار که ترمینال جدیدی باز می‌کنید، اول باید محیط conda را فعال کنید**، بعد برنامه را اجرا کنید:

```powershell
# ۱) رفتن به پوشه‌ی پروژه
cd D:\SC-VAE

# ۲) فعال‌سازی محیط conda
#    (نام محیط بسته به راهی که در مرحله ۴ انتخاب کردید: «scvae» برای راه ۱ یا «torch» برای راه ۲)
conda activate scvae

# ۳) اجرای آموزش
python main-stage1.py `
  --dataset FFHQ `
  --model-config ./configs/ffhq/stage1/ffhq256-scvae16x16.yaml `
  --device cuda `
  --num-workers 0
```

نکات:
- `--device cuda` پیش‌فرض است. اگر CUDA در دسترس نباشد، کد **خودکار روی CPU** می‌افتد (بسیار کند — فقط برای تست منطق، نه آموزش واقعی).
- `--num-workers 0` روی ویندوز امن‌ترین مقدار است (از مشکلات multiprocessing جلوگیری می‌کند).
- آموزش به‌صورت **تک‌پردازشی / تک‌GPU** اجرا می‌شود (پیش‌فرض `world_size=1`). نیازی به راه‌اندازی توزیع‌شده (DDP/NCCL) نیست.

## ۹) عیب‌یابی و مشکلات شناخته‌شده

| مشکل | علت / راه‌حل |
|------|--------------|
| `torch.cuda.is_available()` برابر `False` است | درایور NVIDIA نصب/به‌روز نیست، یا نسخه‌ی CPU پای‌تورچ نصب شده. دستور نصب torch با `+cu116` و `--extra-index-url` را دوباره اجرا کنید. |
| روی این سیستم فعلی GPU نیست | باید روی سیستم دارای کارت NVIDIA اجرا شود؛ وگرنه آموزش روی CPU بسیار کند خواهد بود. |
| خطای `range` در `make_grid` | نسخه‌ی torchvision را روی **0.14.0** نگه دارید (نسخه‌ی جدید این پارامتر را حذف کرده). |
| `FileNotFoundError` برای تصاویر یا فایل لیست | مسیر `dataset/FFHQ/resized/` یا کلید `root` در کانفیگ، یا نام فایل‌ها در `ffhqtrain.txt`/`ffhqvalidation.txt` درست نیست. |
| `CUDA out of memory` | مقدار `batch_size` را در کانفیگ کم کنید (مثلاً 2 یا 4). |
| `conda env create -f environment.yml` سرِ نصب torch خطا می‌دهد | باید خط `--extra-index-url https://download.pytorch.org/whl/cu116` در بخش pip فایل باشد (اکنون اضافه شده). اگر باز هم buildهای پین‌شده resolve نشدند، از «راه ۱» (محیط تمیز + pip) استفاده کنید. |
| `conda` شناخته نمی‌شود | conda نصب نیست یا در PATH نیست؛ Miniconda را نصب و `conda init powershell` را اجرا کنید (مرحله ۴.۱). |
| دنبال `requirements.txt` می‌گردید | چنین فایلی وجود ندارد؛ از conda (مرحله ۴) یا نصب pip استفاده کنید. |
| خطای NCCL / آموزش چند-GPU روی ویندوز | NCCL روی ویندوز پشتیبانی نمی‌شود. حالت پیش‌فرض تک‌پردازشی است و مشکلی ندارد؛ آموزش چند-GPU روی ویندوز ممکن نیست. |
| `ModuleNotFoundError` برای بسته‌ای | همان بسته را با `pip install <نام>` نصب کنید (روش ب فقط بسته‌های اصلی را دارد). |
